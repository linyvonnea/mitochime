\section{Methods}
\label{sec:methods}

\subsection{Overview}
Figure~\ref{fig:pipeline_modeling} summarizes the MitoChime workflow: paired-end read simulation, alignment to a mitochondrial reference, (i) tabular feature extraction for classical machine learning and (ii) sequence-only dataset preparation for deep learning, followed by model inference, pair-safe read filtering, and downstream assembly evaluation.

\subsection{Data simulation and alignment}
Paired-end Illumina reads were simulated from the \textit{Sardinella lemuru} mitochondrial reference (NC\_039553.1). Clean read pairs were generated with \texttt{wgsim} (150\,bp) using an error rate of 0.05. Chimeric templates were constructed by concatenating two non-adjacent reference segments, optionally retaining short overlaps to approximate PCR template switching, and were simulated with the same read-length configuration but with error rate set to 0 to preserve junction signals. Reads were aligned to the reference using \texttt{minimap2} in short-read mode (\texttt{-ax sr}) and processed with \texttt{samtools} to produce sorted and indexed BAM files.

\subsection{Feature and sequence extraction}
Two complementary input representations were prepared from the same labeled read pairs to support both classical machine learning and deep learning.

\paragraph{Tabular feature matrix (classical machine learning).}
For each primary-mapped read, a fixed-length feature vector was computed using \texttt{pysam} and NumPy. Feature families included: (i) supplementary-alignment and alignment-structure summaries derived from SA tags (e.g., presence/count of supplementary alignments, segment distances, strand consistency, and MAPQ/NM aggregates); (ii) CIGAR-derived clipping statistics (e.g., left/right soft clipping, total clipped bases, and an inferred breakpoint position); (iii) k-mer composition discontinuity between read segments ($k=6$), quantified via cosine distance and Jensen--Shannon divergence between left/right k-mer profiles; and (iv) microhomology descriptors around the inferred breakpoint (longest exact suffix--prefix overlap within a $\pm40$\,bp window and its GC fraction), together with basic alignment-quality indicators.

\paragraph{Sequence-only dataset (deep learning).}
In parallel, the raw nucleotide sequence (length $L=150$) was retained as a separate input for the CNN. Sequences were one-hot encoded at training and inference time, with four base channels (A, C, G, T). Ambiguous bases were represented as all zeros.

\subsection{Modeling and evaluation protocol}
The labeled dataset was partitioned into training and test splits (80/20) using stratification to preserve the balanced class ratio. Model selection and tuning were performed on the training split only. Final performance was reported on the held-out test split using accuracy, precision, recall, F1, and ROC--AUC (precision/recall/F1 for the chimeric class).

\paragraph{Classical machine learning (feature-based).}
Feature-based models were implemented in Python using \texttt{scikit-learn}, \texttt{xgboost}, \texttt{lightgbm}, and \texttt{catboost}. An initial benchmark evaluated a panel of classifiers under five-fold stratified cross-validation on the training split, including a dummy baseline, L2-regularized logistic regression, a calibrated linear SVM, k-nearest neighbours, Gaussian Na\"ive Bayes, tree ensembles (Random Forest, ExtraTrees, and bagging with decision trees), and gradient boosting methods (Gradient Boosting, XGBoost, LightGBM, and CatBoost). Based on cross-validation ranking, a subset of competitive model families was carried forward for hyperparameter optimization. Hyperparameters were tuned using randomized search with five-fold stratified cross-validation (\texttt{RandomizedSearchCV}), optimizing chimeric-class F1. Among the feature-based models, a tuned \texttt{GradientBoostingClassifier} (\texttt{sklearn.ensemble}) achieved the best validation ranking and was used as the primary classical model. The hyperparameter search included the number of estimators, \texttt{learning\_rate} (selected value 0.2), and tree-structure constraints; the selected model was refit on the full training split and evaluated once on the held-out test split.
\paragraph{Deep learning (sequence-based).}
A one-dimensional convolutional neural network (CNN1D) was trained using sequence-only input of length $L=150$, represented by one-hot encoding. The architecture comprised three convolutional blocks with 64, 128, and 256 filters (kernel sizes 7, 5, and 3; ReLU activations; max-pooling after the first two blocks), followed by adaptive max pooling and a two-layer MLP head ($256\rightarrow128\rightarrow2$) with dropout (0.2). Training used mini-batches of 128 for up to 30 epochs with learning rate 0.001 and weight decay $10^{-4}$. The selected checkpoint was the model achieving the best held-out F1 during training.

\subsection{Filtering strategy}
Filtering was performed using model scores for the chimeric class and a user-specified decision threshold $\tau\in[0,1]$. For each read pair, the predicted chimeric probability $p$ was computed and pairs were retained if $p < \tau$ (i.e., higher $\tau$ yields stricter chimera removal). Pair-safe filtering was enforced by normalizing identifiers to a base read-pair ID: leading \texttt{@} characters and trailing mate suffixes (\texttt{/1}, \texttt{/2}) were removed, and unique base IDs were expanded to mate-specific identifiers by writing both \texttt{base/1} and \texttt{base/2}. Filtered FASTQ outputs were then produced by retaining only R1 records whose base IDs passed the filter and writing only those R2 records with matching base IDs, ensuring consistent retention/removal of both mates.

\subsection{Assembly and downstream evaluation}
Assemblies were generated from unfiltered and filtered read sets using SPAdes. Assembly graphs were summarized using Bandage graph statistics. Graph complexity was quantified using graph-level metrics including node and edge counts, together with total assembled length, across simulated read depths and chimera rates.
